# Instructions

This database tracks customer orders. Keep answers grounded in the semantic layer.

- "Revenue" means **net revenue**: confirmed orders only, refunds excluded. Prefer
  the `net_revenue` measure over ad-hoc arithmetic.
- Monetary columns are stored in **cents**; divide by 100 to report dollars.
- All timestamps are UTC. Relative windows ("last month") are relative to `now()`.
- Prefer the `monthly_recurring_revenue` trusted asset for any MRR question.

# Schema
## analytics.customers — One row per customer.
synonyms: accounts, users
columns:
- customer_id (bigint) — Primary key. [range: 1..200]
- name (text) — Customer display name.
- email (text) — Contact email.
- region (text) — Sales region. [values: us, emea, apac, latam]
- plan (text) — Subscription tier. [values: free, pro, enterprise]
- signup_date (date) — Date the customer signed up. [range: 2018-06-01..2025-05-31]
join: one_to_many -> analytics.orders ON customers.customer_id = orders.customer_id
## analytics.orders — One row per confirmed order. Excludes carts and draft orders.
synonyms: purchases, transactions, sales
columns:
- order_id (bigint) — Primary key. [range: 1..2000]
- customer_id (bigint) — FK to analytics.customers.customer_id. [range: 1..200]
- amount_cents (bigint) — Order total in cents. Divide by 100 for dollars. [range: 127..310229]
- status (text) — One of 'confirmed','refunded','partial_refund'. [values: confirmed, partial_refund, refunded]
- created_at (timestamp) — Order confirmation time (UTC). [range: 2019-02-02..2026-06-28]
join: many_to_one -> analytics.customers ON orders.customer_id = customers.customer_id
measure net_revenue: SUM(CASE WHEN status='confirmed' THEN amount_cents ELSE 0 END)/100.0  — Revenue net of refunds, in dollars.
filter last_quarter: created_at >= date_trunc('quarter', now()) - interval '3 months'

# Trusted assets (prefer these over ad-hoc SQL; cite the one you use)
## monthly_recurring_revenue(month date) — Official monthly net revenue. Prefer this over ad-hoc revenue math.
SELECT SUM(amount_cents) / 100.0 AS net_revenue
FROM analytics.orders
WHERE status = 'confirmed'
  AND created_at >= date_trunc('month', :month)
  AND created_at <  date_trunc('month', :month) + interval '1 month';

# Examples
Q: What was net revenue last month?
SQL:
SELECT SUM(amount_cents)/100.0 AS net_revenue
FROM analytics.orders
WHERE status='confirmed'
  AND created_at >= date_trunc('month', now()) - interval '1 month'
  AND created_at <  date_trunc('month', now());
Q: How many orders were refunded?
SQL:
SELECT COUNT(*) AS refunded_orders
FROM analytics.orders
WHERE status='refunded';
Q: What is net revenue by region?
SQL:
SELECT c.region,
       SUM(CASE WHEN o.status='confirmed' THEN o.amount_cents ELSE 0 END)/100.0 AS net_revenue
FROM analytics.orders o
JOIN analytics.customers c ON o.customer_id = c.customer_id
GROUP BY c.region
ORDER BY net_revenue DESC;
